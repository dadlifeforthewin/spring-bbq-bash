- API routes follow `src/app/api/stations/activity/route.ts` structure (zod + volunteer-auth + dual-write)

- 8 new glyph SVGs replacing `SparkGlyph` fallback in `StationPicker`

## Database schema

### Migration `0010_prizes.sql`

```sql
create table prizes (
  id          uuid primary key default gen_random_uuid(),
  label       text not null,
  sub         text,                              -- optional secondary label (e.g., "Toy · Kids 5+")
  sort_order  int not null default 0,
  active      boolean not null default true,
  created_at  timestamptz not null default now()
);

create table prize_redemptions (
  id              uuid primary key default gen_random_uuid(),
  child_id        uuid not null references children(id) on delete cascade,
  prize_id        uuid not null references prizes(id)  on delete restrict,
  volunteer_name  text,
  updated_at      timestamptz not null default now(),
  unique (child_id)                              -- one prize per kid; [Change] path upserts
);

create index prize_redemptions_prize_idx on prize_redemptions(prize_id);

create trigger prize_redemptions_touch before update on prize_redemptions
  for each row execute function touch_updated_at();
```

**No seed.** Admin populates via `/admin/prizes` pre-event.

### Migration `0011_cleanup.sql`

```sql
-- Global task catalog; reused across events.
create table cleanup_tasks (
  id          uuid primary key default gen_random_uuid(),
  label       text not null,
  sub         text,                              -- location/detail (e.g., "Main Tent")
  sort_order  int not null default 0,
  active      boolean not null default true,
  created_at  timestamptz not null default now()
);

-- Event-scoped. Toggle on = insert row; toggle off = delete row.
create table cleanup_completions (
  id             uuid primary key default gen_random_uuid(),
  event_id       uuid not null references events(id) on delete cascade,
  task_id        uuid not null references cleanup_tasks(id) on delete cascade,
  completed_by   text,                           -- volunteer name (from request body)
  completed_at   timestamptz not null default now(),
  unique (event_id, task_id)
);

create index cleanup_completions_event_idx on cleanup_completions(event_id);

-- Event-scoped append log. Every CLOSE OUT tap writes a new row.
create table cleanup_locks (
  id          uuid primary key default gen_random_uuid(),
  event_id    uuid not null references events(id) on delete cascade,
  locked_by   text,
  locked_at   timestamptz not null default now()
);

create index cleanup_locks_event_idx on cleanup_locks(event_id);
```

## API contracts

### `POST /api/stations/prize-wheel/redeem`

**Auth:** `isVolunteerAuthed()` gate (401 if not).

**Request body (zod):**
```ts
{
  child_id: uuid,
  prize_id: uuid,
  volunteer_name?: string  // matches existing activity route pattern
}
```

**Behavior:**
1. Load child — 404 if missing; 409 if `!checked_in_at`; 409 if `checked_out_at`.
2. Validate prize — 404 if missing or `!active`.
3. Upsert `prize_redemptions` row keyed on `child_id` (insert or update prize_id + volunteer_name + updated_at).
4. Update `children.prize_wheel_used_at = now()` if null (first time) — **do not overwrite** on re-edit (timestamp is "first redemption time", not "last edit time"). This preserves the existing semantic that other pages depend on.
5. Insert a `station_events` row for the unified activity feed: `station='prize_wheel'`, `event_type='ticket_spend'`, `tickets_delta=0`, `item_name=prize.label`, `volunteer_name`, `notes=null`. On re-edit, insert a second station_events row (audit trail showing the change).
6. Return `{ ok: true, prize: { id, label }, updated: boolean }` where `updated=true` means the row was changed (rescan + change), `false` means first redemption.

### `GET /api/stations/prize-wheel/lookup?child_id=…`

**Auth:** `isVolunteerAuthed()`.

**Purpose:** Station page calls this after QR scan to determine whether to render the chip grid or the "ALREADY REDEEMED" card.

**Response:** `{ child: {id, first_name, last_name, photo_consent_app, prize_wheel_used_at}, redemption: PrizeRedemption|null, prize_label: string|null }`.

### `GET /api/admin/prizes`
List prizes ordered by `sort_order asc, created_at asc`.

### `POST /api/admin/prizes`
Create: `{ label, sub?, sort_order? }`.

### `PATCH /api/admin/prizes/[id]`
Edit: any of `{ label?, sub?, sort_order?, active? }`.

### `DELETE /api/admin/prizes/[id]`
Soft delete via `active=false` (hard delete blocked by `prize_redemptions.prize_id ON DELETE RESTRICT`).

### `POST /api/stations/cleanup/toggle`

**Auth:** `isVolunteerAuthed()`.

**Request:**
```ts
{
  task_id: uuid,
  completed: boolean,
  volunteer_name?: string
}
```

**Behavior:** Look up current event (single-event repo — use `events` table's sole row, or pick by `event_date=today` if multi-event logic exists; confirm during implementation). If `completed=true`: upsert a `cleanup_completions` row on (event_id, task_id). If `completed=false`: delete the row.

**Response:** `{ ok: true, remaining: number, total: number }` — server recounts active tasks and open rows so the UI doesn't have to.

### `GET /api/stations/cleanup/state`

**Auth:** `isVolunteerAuthed()`.

**Response:** `{ tasks: CleanupTask[], completed_task_ids: string[], locked: boolean }`. Station page polls this on mount + after each toggle.

### `POST /api/stations/cleanup/lock`

**Auth:** `isVolunteerAuthed()`.

**Request:** `{ volunteer_name?: string }`.

**Behavior:** Only if `remaining === 0` server-side (recount to avoid race). Insert a new `cleanup_locks` row. Return `{ ok: true, locked_at }`.

### Admin cleanup endpoints
`GET /api/admin/cleanup` · `POST /api/admin/cleanup` · `PATCH /api/admin/cleanup/[id]` · `DELETE /api/admin/cleanup/[id]` — identical shape to prizes endpoints.

## UI specs

### `/station/prize_wheel` (Task 5.5.4)

**Layout:**
1. `<PageHead>` — back to `/station`, title `"PRIZE WHEEL"`, sub `"Scan kid · tap the prize they won."` Right slot: `<Chip tone="gold">GOLD</Chip>` or similar station marker matching the rest of the pages.
2. `<NeonScanner>` bracket frame with `html5-qrcode` — identical pattern to CheckInStation.
3. After scan → state branches on redemption lookup response:

**Branch A (first redemption):** Render chip grid of active prizes (sorted). Each chip is a `<Chip tone="gold" size="lg">` with `label` + optional `sub` on a second line. Layout: 2-column on narrow screens, 3-column ≥ tablet. Tap = POST to redeem → show "LOGGED ✓" affirmation card for 2000ms → auto-clear back to scanner.

**Branch B (already redeemed):** Render a `<SignPanel tone="gold">` with prominent label `"ALREADY REDEEMED"` + the prize label in display font + kid's name + timestamp. Small ghost link `[Change prize]` bottom-right → opens the chip grid (Branch A layout, marked "update mode"). Saving follows the same POST path; returns same 2s affirmation.

**Affirmation card:** Reuses `<SignPanel tone="mint">` + `GlyphGlow` check, centered, 2s fade. Auto-dismisses. No buttons — volunteers are moving fast.

**Empty state:** If no active prizes in DB, show `<SignPanel tone="warn">` with "No prizes configured — open `/admin/prizes` to add them." No scan possible.

## Risks + follow-ups

- **Story-generator prize_won field**: if the story-generation cron has already run for some kids before a late prize redemption, that email won't include the prize. Acceptable for this event (summary email is batched post-event, not real-time). Document as known limitation.
- **Volunteer name on redemption**: matches the existing activity-route pattern (client-supplied). Not forgery-proof, but the volunteer station is behind `isVolunteerAuthed` and the event is a 3-hour in-person affair — not worth hardening.
- **Single-event assumption**: cleanup endpoints resolve `event_id` by picking the single events row. If the repo ever supports multi-event concurrently, the cleanup endpoints need an explicit `event_id` param. Note in the route comments.
- **Prize delete**: soft-delete only (ON DELETE RESTRICT on `prize_redemptions.prize_id`). Admin cannot hard-delete a prize that's been redeemed. UI should reflect this (greyed delete button with tooltip when `active=false` and redemptions exist).
- **Post-merge**: `/dev/glow` dev route still needs to be gated before merging `glow-redesign` into `kid-profile-rebuild` (tracked in STATUS).
